初等数据加密-对称加密算法

发表于 2018-03-04 | 分类于 Java加解密

对于大多数对称加密算法，解密算法是加密算法的逆运算。加密和解密的密钥相同.

1 - 数据加密标准–DES

DES和DESede统称DES系列算法。是对称加密算法领域中的典型算法。DESede算法是对DES算法的三重迭代，增加其算法安全性。

DES算法密钥偏短，只有56位，迭代次数少，受到如查分密码分析和线性密码分析等各种攻击威胁。由于DES算法具有半公开性质。被怀疑是美国安全局安置的后门。

但是，各种对称加密算法都是由DES算法发展而来。

1.1 - 模型分析

1.2 - 实现

密钥长度和安全性成正比。Java6只支持56为密钥长度。而bouncy castle提供64位密钥长度支持。在此基础上配合不同的填充方式。如PKCS5Padding, PKCS7Padding。

密钥的构建主要通过KeyGenetator完成生成操作。

2 - 高级数据加密AES

通过对DES算法的简单改造，得到的DESede算法虽然在一定程度上提升了算法的安全强度。但其低效的加密实现和较慢的处理速度仍不能满足对安全的要求。AES算法正是基于这种缘由诞生。

经过验证，AES算法能够有效的抵御已知的针对DES算法的所有攻击方法。如部分差分攻击，相关密钥攻击等。

AES算法因密钥建立时间短、灵敏性好、内存需求低等优点，在各个领域得到广泛的研究。

发表于 2018-03-04 | 分类于 JVM

3.1 - 掌握跟踪调试参数

虚拟机提供了一些跟踪系统状态的参数，来跟踪系统的运行状态.

3.1.1 - 跟踪垃圾回收 - 读懂虚拟机日志

如果垃圾回收出现频繁，或者占用较长的cpu时间，这时候就需要跟踪参数来进一步甄别垃圾回收的效率。

  1. -XX:+PrintGC : 只要遇到gc, 就会打印日志，如下：

解读：每次gc占用一行。

  1. -XX:+PrintGCDetails : 输出更详细的gc日志

  2. -XX:+PrintHeapAtGC : gc前后分别打印堆的信息

  3. -XX:+PrintGCTimeStamps ： 打印gc发生的时间，是jvm启动后经过的时间。在jvm启动后的0.123秒、0.124秒分别发生了一次gc

  4. -XX:+PrintGCApplicationConcurrentTime : 打印应用程序执行时间

  5. -XX:+PrintGCApplicationStoppedTime : 打印程序停顿时间，Java8下无效

  6. -XX:+PrintReferenceGC : 跟踪系统内软引用、弱引用、虚引用

3.1.2 - 类加载/卸载跟踪

  1. -verbose:class : 跟踪类的加载和卸载

  2. -XX:+TraceClassLoading : 跟踪类的加载

  3. -XX:+TraceClassUnloading : 跟踪类的卸载

3.1.2 - 系统参数查看

目前Java虚拟机支持很多可配置的参数，不同的参数对程序的运行效果可能有很大的影响。有必要明确当前系统的实际运行参数。

  1. -XX:+PrintVMOptions: 打印虚拟机接受到的命令行显示参数

  2. -XX:+PrintCommandLineFlags: 打印虚拟机接受到的命令行显示和隐式参数，隐式参数不一定是通过命令行指定的。可能是虚拟机启动时自行配置的。

  3. -XX:+PrintFlagsFinal: 打印所有系统参数的值

  • 虚拟机尽量维持在初始堆空间范围内运行
  • 如果初始堆空间耗尽，虚拟机就会对堆空间进行扩展。上限为最大堆空间。

最大堆：

  • 通过-Xmx指定

  • -Xmn参数对系统性能以及gc行为有很大影响

3.3 - 别让性能有缺口：了解非堆内存的配置

方法区、线程栈、直接内存，这些区域是和堆内存相对独立的。

3.3.1 - 方法区配置

主要存放类的元数据信息。

  • -XX:MaxMataspaceSize : jdk1.8使用。mataspace只受系统可用内存的限制，但是可以使用这个参数来设置最大值

3.3.2 - 栈配置

  • -Xss : 指定线程的栈大小

3.3.3 - 直接内存配置

在NIO被广泛使用后，直接内存的使用也变得很普遍。

直接内存跳过了Java堆，在一定程度上加快了内存空间的访问速度。但是武断的认为直接内存就一定能加快访问速度也是不正确的。

最大可用直接内存使用参数-XX:MaxDirectMemorySize指定。如果不设置，默认为最大堆内存的值。当直接内存的使用量达到最大值时，就会触发垃圾回收。如果垃圾回收不能有效释放足够空间，直接内存溢出也会引起系统oom.
